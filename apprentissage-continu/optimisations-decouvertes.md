## MÉTHODE D'EXPÉRIMENTATION

### Process Standard Test Optimisation

**Étape 1 : Préparation**
- Définir hypothèse claire
- Établir critères de succès (ex: >X% engagement)
- Planifier 3-5 tests minimum

**Étape 2 : Exécution**
- Appliquer optimisation selon spécifications
- Garder autres variables constantes (timing, thématique)
- Documenter contexte de chaque test

**Étape 3 : Mesure**
- Analyser performance 72h post-publication
- Comparer à baseline (performance moyenne posts standards)
- Noter observations qualitatives (types commentaires, etc.)

**Étape 4 : Analyse**
- Calculer performance moyenne sur tous tests
- Identifier patterns (marche mieux avec quel persona/thématique ?)
- Décider : Valider / Continuer tests / Rejeter

**Étape 5 : Documentation**
- Si validé → Transférer patterns-succes.md
- Si rejeté → Archiver avec learnings
- Si incertain → Continuer tests avec ajustements

### Critères de Validation

**Pour passer de 🔬 En test → ⚡ Prometteur** :
- 3 tests réalisés
- Performance > baseline +15%
- Feedback qualitatif positif

**Pour passer de ⚡ Prometteur → ✅ Validé** :
- 5+ tests réalisés
- Performance > baseline +20% constante
- Taux succès >80%
- Réplicable dans contextes variés

**Pour passer à ❌ Rejeté** :
- 3+ tests avec performance < baseline
- OU feedback qualitatif négatif récurrent
- OU effort/résultat non soutenable
