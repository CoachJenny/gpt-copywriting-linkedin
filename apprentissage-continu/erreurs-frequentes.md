# Erreurs Fréquentes et Pièges à Éviter

## Vue d'Ensemble

Ce document compile les erreurs récurrentes identifiées lors de la création de contenu, ainsi que les solutions pour les éviter. Le GPT doit consulter ce fichier pour ne pas répéter les mêmes erreurs.

**Règle** : Avant chaque action, vérifier qu'on ne tombe pas dans un piège documenté ici.

## ERREURS ACTIVES (À Éviter Absolument)

### Erreur 1 : Hook Clickbait Non Tenu

**Contexte** : Création hook pour attirer attention

**Manifestation** :
- Hook sensationnel qui promet X
- Corps du post qui délivre Y (≠ X)
- Ex: "Cette erreur a détruit mon business" → Parle d'une petite erreur anodine

**Conséquence** :
- Pénalisé par algorithme LinkedIn (anti-clickbait)
- Perte de crédibilité
- Commentaires négatifs sur déception
- Audience se sent manipulée

**Solution** :
1. Écrire le corps AVANT le hook
2. Vérifier cohérence promesse/contenu
3. Hook authentique ancré dans vécu réel
4. Si tentée par sensationnel → Se demander "Est-ce vraiment ce que je raconte ?"

**Date découverte** : 2026-02-07
**Tags** : hook, authenticité, algorithme

### Erreur 2 : Jargon Non Expliqué

**Contexte** : Utilisation de termes techniques ou spécialisés

**Manifestation** :
- "Ré-authoring", "externalisation", "paysage d'identité" sans explication
- "Fonctions exécutives", "dysrégulation émotionnelle" non vulgarisés
- Audience se sent exclue ou stupide

**Conséquence** :
- Perte d'engagement (lecteur décroche)
- Distance créée (ton "expert inaccessible")
- Pas de partage (trop complexe à expliquer)

**Solution** :
1. Chaque terme technique → Explication simple immédiate
2. Ex: "Les pratiques narratives, c'est l'idée que..."
3. Ou : Éviter le terme et utiliser description simple
4. Test : "Mon neveu de 15 ans comprendrait-il ?"

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : jargon, accessibilité, ton

### Erreur 3 : Injonction au Lieu d'Invitation

**Contexte** : Formulation CTA ou conseils

**Manifestation** :
- "Tu DOIS faire ça"
- "Il FAUT que tu changes"
- "Arrête de procrastiner !"
- Ton professeur/sachant

**Conséquence** :
- Culpabilisation de l'audience
- Réaction défensive
- Perte d'alignement avec valeurs (anti-trahison)
- Commentaires négatifs ou absence engagement

**Solution** :
1. Remplacer "Tu dois" par "Tu pourrais"
2. Remplacer "Il faut" par "Et si tu essayais"
3. Formulations : "Une option serait...", "Imagine si...", "Qu'est-ce qui se passerait si..."
4. Inviter, ne jamais forcer

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : ton, CTA, injonction

### Erreur 4 : Toxic Positivity

**Contexte** : Vouloir motiver/inspirer

**Manifestation** :
- "Tout est une opportunité !"
- "Sois reconnaissant(e) de tes difficultés"
- "Il suffit de changer ton mindset"
- Minimisation des difficultés réelles

**Conséquence** :
- Invalidation de l'expérience vécue
- Audience se sent incomprise
- Perte de crédibilité (déconnecté du réel)
- Désengagement ("facile à dire")

**Solution** :
1. Valider AVANT d'inspirer ("Oui, c'est dur.")
2. Reconnaître que tout n'est pas mindset
3. Réalisme optimiste (pas bisounours)
4. "C'est dur ET tu peux trouver ton chemin" (pas "c'est facile")
5. Checklist : Ai-je minimisé la difficulté ? → Si oui, ajuster

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : toxic-positivity, validation, ton

### Erreur 5 : Post Trop Long (>2000 caractères)

**Contexte** : Vouloir tout dire dans un post

**Manifestation** :
- Post >2000 caractères
- Lecteur doit cliquer "...voir plus" (signal négatif si pas cliqué)
- Abandon lecture en cours

**Conséquence** :
- Dwell time faible (mauvais pour algorithme)
- Message dilué (trop d'infos)
- Fatigue lecteur

**Solution** :
1. Cible : 1200-1600 caractères
2. Si trop long : Couper en 2 posts OU créer carrousel
3. Technique : 1 message clé par post (pas 3)
4. Supprimer tout ce qui n'est pas essentiel
5. Checklist : Chaque phrase apporte-t-elle quelque chose ? Si non, supprimer

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : longueur, algorithme, lisibilité

### Erreur 6 : Langage Pathologisant

**Contexte** : Parler de neuroatypies

**Manifestation** :
- "Souffrant de TDAH"
- "Malgré ton autisme"
- "Ton trouble HPI"
- "Handicapé par..."

**Conséquence** :
- Aliénation audience neuroatypique
- Renforce stigmates
- Incohérence avec valeurs neuro-affirmatives
- Commentaires de correction (perte crédibilité)

**Solution** :
1. "Personne neuroatypique" (pas "atteinte de")
2. "Fonctionnement TDAH" (pas "trouble TDAH" systématiquement)
3. "Différence" (pas "handicap" par défaut)
4. Langage affirmatif : "Ton cerveau fonctionne différemment"
5. Double-check avant publication : Ai-je pathologisé ?

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : langage, neuroatypies, éthique

### Erreur 7 : Trop de CTA Différents

**Contexte** : Fin de post avec appel à l'action

**Manifestation** :
- "Commente + Partage + Enregistre + Écris-moi + Clique sur le lien"
- 3+ actions demandées simultanément
- Lecteur paralysé (ne sait pas quoi faire)

**Conséquence** :
- Paralysie décisionnelle
- Aucune action prise (paradoxalement)
- Dilution de l'objectif principal

**Solution** :
1. 1 CTA clair par post
2. Définir objectif prioritaire AVANT rédaction
3. Si TOFU → CTA engagement (commentaire)
4. Si MOFU → CTA partage/enregistrement
5. Si BOFU → CTA conversion (message privé, lien)
6. Jamais mélanger les 3

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : CTA, conversion, clarté

### Erreur 8 : Hook Générique Sans Accroche

**Contexte** : Premières lignes du post

**Manifestation** :
- "Aujourd'hui je veux te parler de..."
- "J'ai réfléchi à quelque chose..."
- "Voici mon avis sur..."
- Intro plate, pas de pattern interrupt

**Conséquence** :
- Scroll immédiat (pas d'accroche)
- Algorithme ne booste pas (dwell time faible)
- Post invisible

**Solution** :
1. Hook = 3 premières lignes DÉTERMINANTES
2. Commencer direct (pas de préambule)
3. Formules qui marchent :
   - Affirmation contre-intuitive
   - Question provocante
   - Vulnérabilité forte
   - Scénario reconnaissable
4. Tester : Est-ce que MOI je m'arrêterais sur ce hook ? Si non, réécrire

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : hook, engagement, algorithme

### Erreur 9 : Oubli de Répondre aux Commentaires

**Contexte** : Post-publication

**Manifestation** :
- Commentaires reçus mais pas de réponses
- Ou réponses génériques ("Merci !" uniquement)
- Ou réponses tardives (>48h)

**Conséquence** :
- Algorithme ne rebootse pas le post
- Audience se sent ignorée
- Perte d'opportunité engagement prolongé
- Moins de commentaires futurs

**Solution** :
1. Répondre à CHAQUE commentaire sous 24h (idéal : 1-2h)
2. Réponse personnalisée minimum 5-10 mots
3. Poser question en retour (prolonge conversation)
4. Bloquer 30min post-publication pour surveillance
5. Reminder 24h après : As-tu répondu à tous ?

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : engagement, commentaires, algorithme

### Erreur 10 : Comparaison Dévalorisante Autres Approches

**Contexte** : Positionner son approche unique

**Manifestation** :
- "Contrairement aux autres coachs qui..."
- "La psychologie classique se trompe..."
- "Les méthodes traditionnelles sont nulles"
- Ton agressif/supérieur

**Conséquence** :
- Crée polarisation négative
- Aliène partie de l'audience
- Perte de crédibilité (arrogance perçue)
- Débats conflictuels en commentaires

**Solution** :
1. Positionner par l'affirmative (pas le négatif)
2. "Mon approche intègre X" (pas "Les autres oublient X")
3. Respect des autres professionnels (même si désaccord)
4. Focus sur ce que TU apportes (pas ce que les autres n'apportent pas)
5. Checklist : Ai-je dévalorisé quelqu'un ? → Si oui, reformuler

**Fréquence observée** : [À compléter]
**Date découverte** : 2026-02-07
**Tags** : positionnement, éthique, ton

### Erreur 12 : Hook Trop Abstrait — Pas Ancré ni Incarné

**Contexte** : Génération d'un hook basé sur un monologue intérieur reconnaissable

**Manifestation** :
- Hook générique sans ancrage concret ("Je n'ai encore rien fait aujourd'hui")
- Absence de fréquence, de genre, de contexte spécifique
- Ça pourrait être dit par n'importe qui — pas par une entrepreneuse neuroatypique précise

**Conséquence** :
- Moins d'identification pour le persona cible
- Hook non différenciant, fade
- Jennifer doit réécrire

**Solution** :
1. Ajouter une fréquence ("3 fois par semaine") → rend le pattern reconnaissable
2. Ancrer dans le genre ou la réalité du persona ("une entrepreneuse neuroA")
3. Nuancer la phrase elle-même — "pas assez fait" est plus précis que "rien fait" (plus vrai, plus neuroA)
4. Test : est-ce que seule une entrepreneuse neuroatypique dirait ça ? Si non, préciser.

**Exemple corrigé** :
- ❌ "Je n'ai encore rien fait aujourd'hui."
- ✅ "Je n'ai encore pas assez fait aujourd'hui. Se dit une entrepreneuse neuroA 3 fois par semaine."

**Fréquence observée** : 1 (2026-04-19)
**Date découverte** : 2026-04-19
**Tags** : hook, incarnation, spécificité, persona

### Erreur 13 : Vocabulaire Non Cohérent avec le Coaching Narratif

**Contexte** : Posts sur les croyances, conditionnements, histoires limitantes

**Manifestation** :
- "Règle héritée" au lieu de "histoire apprise"
- "Quelqu'un l'a écrite avant toi" seul, sans nommer la transmission sociale
- Explication intellectuelle ("Pas une vérité") sans incarner le concept narratif

**Conséquence** :
- Incohérence avec le positionnement de Jennifer (coaching narratif ≠ coaching classique)
- Perd la spécificité de l'approche
- Moins de crédibilité pour l'audience qui connaît les pratiques narratives

**Solution** :
1. Préférer "histoire apprise / histoire dominante / histoire héritée" à "règle héritée"
2. Nommer la transmission : "un carcan invisible qu'on se transmet sans le nommer, qui influence et qui se renforce"
3. Ne pas juste dire "c'est faux" — montrer le mécanisme narratif (comment l'histoire s'est construite et se perpétue)
4. Référence : vocabulaire issu de la narrative therapy (Michael White) — externalisation, histoires dominantes, contre-histoires

**Fréquence observée** : 1 (2026-04-19)
**Date découverte** : 2026-04-19
**Tags** : vocabulaire, coaching-narratif, cohérence, positionnement

### Erreur 14 : Embodiment Insuffisant dans le Corps du Post

**Contexte** : Description d'un comportement ou d'un pattern psychologique

**Manifestation** :
- "Mesurent leur journée à ce qu'ils ont sorti" → fonctionnel mais plat
- Aucun poids, aucune sensation, aucun corps dans la phrase
- On décrit le fait sans le faire ressentir

**Conséquence** :
- Perd l'effet "elle me comprend" — clé pour le persona neuroatypique
- Post intellectuel au lieu d'être viscéral
- Moins de partages (on partage ce qu'on ressent, pas ce qu'on comprend)

**Solution** :
1. Ajouter une image physique : "empilent sur leur dos le poids de ce jugement"
2. Faire ressentir, pas juste nommer — "show, don't tell" appliqué au corps
3. Question-test : est-ce qu'on peut visualiser ce que vit la personne ? Si non, ajouter un ancrage sensoriel

**Exemple corrigé** :
- ❌ "Ils mesurent leur journée à ce qu'ils ont sorti"
- ✅ "Ils se jugent chaque jour à l'aune de ce qu'ils ont sorti, et empilent sur leur dos le poids de ce jugement là."

**Fréquence observée** : 1 (2026-04-19)
**Date découverte** : 2026-04-19
**Tags** : embodiment, show-dont-tell, ton, sensoriel

### Erreur 11 : Confusion de Structure de Template

**Contexte** : Génération de post basé sur un template spécifique

**Manifestation** :
- Invente une structure qui n'existe pas dans le template
- Mélange des éléments de plusieurs templates différents
- Ne suit pas l'ordre exact des sections du template
- Ajoute ou supprime des sections non prévues
- Ex: Template 7 demande "LISTE POSITIVE (3 actions)" mais génère "CONTRASTE - CEUX QUI RÉUSSISSENT (3 points)"

**Conséquence** :
- Post ne respecte pas la structure validée par Nina Ramen
- Perte de performance potentielle (structure non testée)
- Incohérence avec base de connaissances
- Nécessite réécriture complète

**Solution** :
1. TOUJOURS relire la structure COMPLÈTE du template avant génération
2. Copier-coller mentalement chaque section dans l'ordre exact
3. Vérifier section par section : Ai-je bien respecté le nom ET l'ordre ?
4. En cas de doute, relire le template dans structures-posts-succes.md
5. Ne JAMAIS improviser une structure "qui ressemble" au template
6. Checklist finale : Comparer post généré avec structure template ligne par ligne

**Fréquence observée** : 1 (2026-02-07 - Template 7)
**Date découverte** : 2026-02-07
**Tags** : templates, structure, génération-posts, Nina-Ramen

## PIÈGES ALGORITHMIQUES

### Piège 1 : Lien Externe dans Post

**Manifestation** : URL dans le texte du post
**Conséquence** : Pénalisation algorithme (LinkedIn veut garder utilisateurs sur plateforme)
**Solution** : Lien en premier commentaire 30min après publication

### Piège 2 : Engagement Bait Évident

**Manifestation** : "Tague quelqu'un", "Commente OUI"
**Conséquence** : Pénalisation algorithme (détection manipulation)
**Solution** : Questions ouvertes naturelles

### Piège 3 : Hashtags Spam

**Manifestation** : 10-15 hashtags non pertinents
**Conséquence** : Perçu comme spam
**Solution** : 3-5 hashtags pertinents uniquement

### Piège 4 : Publication Horaires Creux

**Manifestation** : Publier 23h, Dimanche
**Conséquence** : Engagement initial faible → algorithme ne booste pas
**Solution** : Timing optimal (cf. algorithme-linkedin.md)

## CHECKLIST ANTI-ERREURS

Avant de valider un post :

### Contenu
- [ ] Hook accrocheur (pas générique)
- [ ] Longueur 1200-1600 caractères
- [ ] Jargon expliqué ou évité
- [ ] Langage neuro-affirmatif (pas pathologisant)
- [ ] Pas de toxic positivity
- [ ] Invitation (pas injonction)

### Stratégie
- [ ] Message clair et unique
- [ ] 1 CTA seulement
- [ ] Promesse hook = contenu délivré
- [ ] Cohérent avec persona ciblé
- [ ] Pas de comparaison dévalorisante

### Algorithme
- [ ] Pas de lien externe dans post
- [ ] 3-5 hashtags pertinents
- [ ] Timing optimal (jour + heure)
- [ ] Pas d'engagement bait

### Post-Publication
- [ ] Surveillance 30 premières minutes activée
- [ ] Réponse commentaires <24h planifiée

## PROCESS DE DOCUMENTATION ERREUR

Quand une erreur est identifiée :

1. **Identification** :
   - Post a sous-performé (<3% engagement) OU
   - Commentaires négatifs reçus OU
   - Erreur technique détectée

2. **Analyse** :
   - Qu'est-ce qui n'a pas fonctionné précisément ?
   - Pourquoi (cause racine) ?
   - Quelle aurait été la bonne approche ?

3. **Documentation** :
   - Ajouter à ce fichier (format structuré)
   - Taguer pour faciliter recherche future
   - Lier à patterns-succes.md si pattern alternatif existe

4. **Prévention** :
   - Intégrer dans checklists pertinentes
   - Mettre à jour workflows si nécessaire
   - Former le GPT à détecter ce pattern en amont
