## ANALYSE CROSS-PATTERNS

### Éléments Transversaux qui Fonctionnent Toujours

**Hooks** :
- Affirmation contre-intuitive + "Faux"
- Vulnérabilité authentique (situation difficile)
- Question qui challenge standards
- Scénario hyper-reconnaissable

**Corps** :
- Storytelling avec arc de transformation
- Listes numérotées (3-5 points)
- Alternance phrases courtes/longues (rythme)
- Métaphores du quotidien

**CTAs** :
- Questions ouvertes personnelles ("Et toi ?")
- Invitation à partager expérience
- Pas d'injonction, invitation douce

**Ton** :
- Empathie + Validation
- Anti toxic positivity
- Langage neuro-affirmatif
- Posture d'égal à égal

### Combinaisons Gagnantes

**Combo 1 : Vulnérabilité + Déconstruction**
- Partage galère personnelle
- Déconstruction de ce qu'on croyait à l'époque
- Nouvelle vérité découverte
- Application pour lecteur
→ Résultat : Connexion + Repositionnement

**Combo 2 : Question + Storytelling Court**
- Question provocante
- Mini-story personnel pour illustrer
- Retour à la question
→ Résultat : Engagement profond

**Combo 3 : Liste + Reconnaissance**
- Liste signes/erreurs/astuces
- Chaque point = validation ("C'est normal de...")
- Conclusion déculpabilisante
→ Résultat : Partages + Enregistrements

## PROCESS D'AJOUT DE NOUVEAU PATTERN

### Étapes

1. **Identification** :
   - Post a généré >8% engagement
   - OU résolu problème récurrent de manière innovante
   - OU demandé 3x+ par utilisateur

2. **Documentation** :
   - Remplir template pattern
   - Ajouter en section "Patterns en Test"
   - Définir critères de validation

3. **Validation** :
   - Tester 3-5 fois dans contextes variés
   - Mesurer performance moyenne
   - Si >80% succès → Passer en "Actifs"
   - Si <60% succès → Archiver avec analyse

4. **Intégration** :
   - Ajouter aux workflows si pertinent
   - Mettre à jour structures-posts-succes.md
   - Documenter dans changelog.md

5. **Suivi** :
   - Tracker utilisation mensuelle
   - Réévaluer performance trimestrielle
   - Archiver si déclin confirmé
